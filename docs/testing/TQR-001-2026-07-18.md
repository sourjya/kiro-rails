# Test Quality Review: kiro-rails

## TQR-001 — 2026-07-18

### Scan Manifest
- **Date:** 2026-07-18
- **Scope:** All executable scripts in `scripts/` (15 files)
- **Test framework:** bash (no formal test runner — scripts tested manually via `run_and_watch`)
- **Fixtures:** `tests/fixtures/bug-scribe/` (7 files)
- **Formal test files:** 0

---

### Executive Summary

kiro-rails is a **template/tooling project** — its "source code" is shell scripts and hook definitions, not application code. It has **zero formal test files** (`tests/unit/` is empty). Testing has been done interactively via `run_and_watch` during development but no automated test suite exists that can be run with `pytest` or `bats`.

**Verdict:** MEDIUM risk. The scripts work (proven by manual testing during development), but there's no regression safety net. A future change to `bug-scribe.sh` could break detection without anyone knowing until it fails in the wild.

### Findings

| # | Severity | Finding | Remediation |
|---|----------|---------|-------------|
| 1 | MEDIUM | **No automated test suite.** `tests/unit/` is empty. Scripts were tested interactively but have no regression safety. | Add `bats` (Bash Automated Testing System) test files for the 3 HIGH-risk scripts |
| 2 | MEDIUM | **`lib/template.sh` has no tests.** This is the shared library used by bug-scribe and all future scaffolding. A regression here breaks everything downstream. | Create `tests/unit/test_template_sh.bats` with: single-line substitution, multi-line injection, missing file handling, portable sed |
| 3 | MEDIUM | **`bug-scribe.sh` fixtures exist but no test runner invokes them.** The 7 fixture files are used for manual testing but nothing runs them in CI or on `./scripts/test.sh`. | Create `tests/unit/test_bug_scribe.bats` that exercises all fixtures automatically |
| 4 | LOW | **Detection scripts have no negative test cases.** `empty-catch-detect.sh` was tested with one fixture but there's no file proving it does NOT fire on valid error handling. | Add negative fixtures (correct code that should produce no output) |
| 5 | LOW | **`test-file-scaffold.sh` has no test proving it skips correctly.** Skip patterns (tests/, scripts/, migrations/) were verified interactively but not captured. | Add test cases for skip patterns |

---

### Recommendation

**Add `bats-core` as a dev dependency** and create test files for the three HIGH-risk scripts:

```
tests/
├── unit/
│   ├── test_bug_scribe.bats          # All 7 fixtures + resolve + idempotency
│   ├── test_template_sh.bats         # Render single/multi-line, portability
│   └── test_test_file_scaffold.bats  # Python + TS paths, skip patterns, idempotency
├── fixtures/
│   ├── bug-scribe/         # ✅ already exists (7 files)
│   ├── empty-catch/        # add: valid.py (no finding) + invalid.py (finding)
│   └── deprecated/         # add: valid.py + invalid.py
```

Run with: `bats tests/unit/`

**Priority:** Not blocking (kiro-rails has no CI pipeline — it's a template project). But recommended before v1.0 or before contributors touch the scripts.
