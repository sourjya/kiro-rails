# Maintainability Review: kiro-rails

## MRR-001 — 2026-07-18

### Scan Manifest
- **Date:** 2026-07-18
- **Scope:** `scripts/` (15 files, ~1800 lines), `scripts/lib/` (1 file), `.kiro/hooks/` (30 hooks)
- **ADRs reviewed:** ADR-001 (git discipline), ADR-002 (triangulation model)
- **Context:** kiro-rails is a template project (steering + hooks + scripts), not application code

---

## A. Executive Summary

kiro-rails' scripts are well-documented, consistently structured, and follow a clear pattern. The codebase is small (~1800 lines of bash) and mostly new (built in one session), so it hasn't yet accumulated the typical drift. The main maintainability risks are:

1. **Repeated boilerplate** across the 4 detection scripts (same preamble, same file/ext extraction, same output pattern)
2. **No shared "detection script" framework** — each script re-implements the same structure
3. **`bug-scribe.sh` is growing large** (600+ lines) and could benefit from decomposition
4. **Inconsistent use of `template.sh`** — only `bug-scribe.sh` and `test-file-scaffold.sh` source it; others don't need it but also don't use the portable sed helper they'd benefit from

The strongest quick win: extract a shared preamble function for detection scripts.

## D. Patch Proposals (Top 3)

### 1. `scripts/lib/detect.sh`

```bash
#!/usr/bin/env bash
# scripts/lib/detect.sh — Shared preamble for file-scanning detection scripts

detect_init() {
    FILE="${1:-}"
    [ -z "$FILE" ] && exit 0
    [ -f "$FILE" ] || exit 0
    EXT="${FILE##*.}"
}
```

Usage in detection scripts:
```bash
source "${SCRIPT_DIR}/lib/detect.sh"
detect_init "$1"
# $FILE and $EXT are now set, early-exit handled
```

### 2. `scripts/lib/detect-function.sh`

Move the `detect_function()` and all `FUNC_PATTERN_*` variables from `bug-scribe.sh` into this file. Bug Scribe sources it. Future hooks (e.g., a function-complexity checker) can too.

### 3. `_portable_sed_i` in `lib/template.sh`

```bash
_portable_sed_i() {
    local file="$1"; shift
    if sed --version 2>/dev/null | grep -q GNU; then
        sed -i "$@" "$file"
    else
        sed -i '' "$@" "$file"
    fi
}
```

## Verification Pass

All findings re-examined against ADRs:
- ADR-002 (triangulation) explicitly says each hook is independently disablable — this justifies the "each script is self-contained" pattern even if it means some duplication.
- The detection scripts' identical preambles ARE still worth extracting (Finding #1) because it's pure boilerplate, not intentional independence.
- Finding #4 (changelog grouping) left as "no action" because bash doesn't have better idioms for this and the abstraction would be harder to read than the repetition.
